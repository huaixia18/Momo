# MOMO 开发计划（分阶段）

> 目标：在现有 MVP scaffold 基础上，按阶段把“可演示”升级到“可迭代开发”。

## 阶段 0：基线盘点（已完成）
- 已有小程序单页流程：输入情绪 -> 生成料理 -> 投喂 -> 回复。
- 已有规则引擎与基础单测。

## 阶段 1：领域模型重构（本轮）
### 目标
把情绪食物、MBTI 角色、稀有度规则从散落逻辑升级为“可配置模型”。

### 交付
- 抽离 `foodCatalog` 与 `petProfiles` 数据模型。
- 引擎支持根据输入内容稳定生成食物（可复现）。
- 增加“输入来源”概念（text / image）。

### 验收标准
- 相同输入每次得到同一 food card。
- 单测覆盖 emotion / rarity / deterministic mapping。

---

## 阶段 2：核心玩法增强（本轮）
### 目标
补齐 PRD 中“观察日记 + 成长约束”最小闭环。

### 交付
- 增加喂食日志（session 内）。
- 增加“日记生成器”：由当天喂食日志生成宠物第一人称日记。
- 增加饱食度自然衰减函数（按小时）。

### 验收标准
- 有喂食记录时可生成日记。
- 空记录时给出友好提示。
- 数值更新在 0~100 范围内。

---

## 阶段 3：前端交互升级（本轮）
### 目标
让页面覆盖“输入-投喂-日记”完整链路，并增强状态可读性。

### 交付
- 首页增加输入模式切换（文本/图片描述占位）。
- 增加喂食历史列表。
- 增加“生成观察日记”区域与文案展示。
- 增加“按时间衰减饱食度”触发按钮用于演示。

### 验收标准
- 可视化看到最近喂食记录。
- 一键生成日记，内容与记录相关。
- 演示按钮可触发饱食度变化。

---

## 阶段 4：测试与质量（本轮）
### 目标
确保引擎逻辑和页面脚本达到可维护水平。

### 交付
- 扩展单测：日记生成、衰减逻辑、边界值。
- 语法检查 + 回归测试。

### 验收标准
- `node tests/momoEngine.test.js` 全通过。
- 页面/引擎脚本 `node --check` 无报错。
