# 产品需求文档（PRD）- MOMO 情绪饲养员 v1.0

| 项目名称 | MOMO 情绪饲养员（MOMO: Your Emotional Keeper） |
| :--- | :--- |
| **版本号** | MVP 1.0（最小可行性产品） |
| **产品定位** | AIGC 驱动的情绪养成轻游戏 |
| **核心价值** | 将原本难以消化的情绪，具象化为数字宠物的“食物”，在投喂与陪伴中获得治愈。 |
| **目标平台** | 微信小程序 |
| **适用人群** | 18-35 岁，有高频情感表达需求、喜欢玄学/MBTI、追求仪式感的小红书/泛二次元用户。 |

---

## 一、产品背景与核心概念

用户在日常生活中面临两类情绪痛点：

1. **负面情绪无处宣泄：** 吐槽怕被杠，朋友圈不敢发。
2. **正面瞬间无人分享：** 精心打扮或拍了美景，渴望高质量反馈。

### 核心隐喻（Metaphor）

- **MOMO（宠物）** = 用户的另一个自我 / 理想闺蜜。
- **情绪（输入）** = 宠物的**食物**。
- **AI 回复（输出）** = 宠物进食后的**消化反应**（吐槽/夸赞）。
- **日记** = 情感账单，也是社交货币。

## 三、详细功能需求

### 模块 1：领养与初始化（Onboarding）

**优先级：P0**

**功能描述：** 用户首次进入，需建立与宠物的连接。

**交互逻辑：**

1. **MBTI 选择：** 提供 4 种基础性格供选择（简化版 MBTI）。
   - *快乐修勾（ENFP）：* 热情、治愈、无脑夸。
   - *毒舌猫咪（INTJ）：* 理智、犀利、嘴替。
   - *温柔兔兔（ISFJ）：* 敏感、共情、安慰。
   - *发疯水母（ENTP）：* 搞怪、阴阳怪气、玩梗。
2. **起名：** 给 MOMO 起个名字。
3. **生成：** 破壳动画，宠物诞生。

### 模块 2：情绪料理厨房（The Core Input）

**优先级：P0**

> 这是产品核心玩法，对应“万物拍拍”的拍照生成物品。

**功能描述：** 将非结构化情绪（图/文）转化为结构化游戏道具（食物）。

**输入支持：**

- **文本模式：** 输入框（支持粘贴聊天记录）。
- **视觉模式：** 拍照/相册上传。

**AI 处理逻辑（后端）：**

- **语义/视觉分析：** 识别情绪极性（正向/负向/中性）和具体内容。
- **物品映射算法：**
  - 负面情绪（如愤怒）→ 生成 **“怪味料理”**（例：红烧甲方大饼、酸柠檬气泡水）。
  - 正面情绪（如自拍）→ 生成 **“治愈甜点”**（例：绝美落日舒芙蕾、自信彩虹塔）。
- **稀有度系统（R/SR/SSR）：** 根据描述丰富度或图片精美度，随机赋予稀有度。SSR 级食物有特效光圈。

**输出展示：** 弹出一张卡片，包含【食物像素图】+【风趣描述文案】。

### 模块 3：投喂与共鸣（The Core Interaction）

**优先级：P0**

**功能描述：** 喂食动作触发 AI 情感反馈。

**交互逻辑：**

1. 用户将生成的“料理卡片”拖拽给 MOMO。
2. **进食动画：** MOMO 吃掉食物（咀嚼音效）。
3. **即时反馈（气泡框）：** 宠物根据 MBTI 性格 + 食物内容回应。
   - *场景示例（喂“红烧甲方”给毒舌猫）*
   - *猫咪回应：* “这肉质又老又柴，下次建议直接掀桌子，别把自己气坏了。这种老板留着过年吗？”
4. **保存/分享：** 用户可将【宠物回应截图】一键保存。

### 模块 4：MOMO 的观察日记（Retention & Share）

**优先级：P1**

> 对应“万物拍拍”的宠物视角日记，是留存和裂变关键。

**功能描述：** 系统每天（或用户主动触发）基于当天投喂记录，生成一篇日记。

**内容结构：**

- **视角：** 宠物第一人称（“今天我的两脚兽好像...”）。
- **内容：** 串联当天情绪事件，并升华主题（安慰或鼓励）。
- **配图：** AI 自动生成与当天情绪色调相符的氛围插画。

**样式：** 拍立得或手账风长图，底部带小程序码。

### 模块 5：数值与成长体系（Gamification）

**优先级：P1**

**三维数值：**

1. **饱食度（Hunger）：** 随时间自然衰减，限制无限刷（防沉迷/控制 token 成本），喂食恢复。
2. **亲密度（Bond）：** 喂食累积，达到等级解锁新皮肤或新语录。
3. **心情值（Mood）：** 随喂食内容变化，影响表情待机状态。

**货币（情绪结晶）：**

- 获取：签到、喂食成功、分享日记。
- 消耗：购买宠物小配饰（如墨镜、蝴蝶结）。

## 四、UI/UX 设计风格指南

- **整体调性：** **像素风（Pixel Art）** 或 **Y2K 矢量插画风**（小红书流行且开发成本低）。
- **色彩心理学：**
  - 背景色随时间变化（白天暖白、晚上深蓝）。
  - 关键按钮（投喂）使用多巴胺配色（高饱和粉/绿）。
- **微交互：**
  - 强调**触感反馈（Haptic）**：喂食成功时手机震动。
  - 用拟人化文案替代“加载中”，如“MOMO 正在闻味道...”“MOMO 正在思考怎么怼...”。

## 五、技术架构与 AI 策略（Technical Specs）

### 1）技术栈

- **前端：** 微信小程序原生 或 Uni-app。
- **后端：** 微信云开发（CloudBase）- Serverless 架构，省运维成本。
- **数据库：** 云开发数据库（NoSQL）。

### 2）AI 模型调用策略

- **文本模型（LLM）：**
  - 推荐：**DeepSeek** 或 **Moonshot（Kimi）** API。
  - *System Prompt 核心逻辑：* “你是一只 [MBTI 类型] 的电子宠物。用户给你投喂了 [食物名称]，这代表了 [情绪类型]。请用 [性格关键词] 的语气，简短地（50 字以内）回复用户，旨在提供情绪价值。”
- **图像识别（Vision）：**
  - 调用 GPT-4o-mini 或阿里云通义千问 VL 进行图片标签提取。
- **生图（Image Gen）：**
  - *成本控制方案：* MVP 阶段**不建议**实时生成像素图（成本高、速度慢）。
  - *替代方案：* 预制 50-100 张像素食物素材库 + 20 张背景图，AI 仅匹配素材 ID。

---

## 六、埋点与数据指标（Data Metrics）

用于验证 MVP 成功与否的关键指标：

1. **投喂率：** 日活用户中，完成至少一次“制作料理 + 投喂”的比例。
2. **分享率（K-Factor）：** “日记”或“神回复卡片”被保存/分享次数。
3. **次留（Retention）：** 第二天回访 MOMO 的用户比例。

---

## 七、开发排期建议（4 周 MVP）

- **Week 1（设计与素材）：** 确定 UI 风格，绘制宠物 3 种状态（开心/普通/emo）和 20 种基础食物素材。
- **Week 2（核心逻辑）：** 完成“图文识别 → 食物映射 → LLM 回复”的后端链路。
- **Week 3（前端交互）：** 完成拖拽喂食动画、日记生成页面。
- **Week 4（调试与上线）：** Prompt 调优（避免机器感）、接入内容安全接口、上线发布。
